When you click on the search button the plugin call the "searchfrm(context)". This method have a parameter "context" this is an istance of the internal context. You will compose the query by calling (the method below) on this object. When the "searchfrm(context)" return the query start. Below a list of the context's method.

clearSearch() - Init the plugin serch.

from(resturl) - Specify the source data as a rest url. 

pagingWithSize(value) - This method tell the plugin that you want a paging result set.

where(expression, param) - Specify the where clause in string format "City=@ and Country=@1", param are an array of param. 

orderBy(expression) - Specify the orderby clause in string format "CustomerID".

select(expression) - Specify the select clause in string format "new(CustomerID,City,Country,Orders)". This parameter is not require. If you
do not specific it the query take all data.

take(value) - Specifiy the take value. If you use this method the paging function will be disabled. This is the opposite of pagingWithSize(value).

skip(value) - Specifiy the skip value. If you use this method the paging function will be disabled. This is the opposite of pagingWithSize(value).

applyTempClient(value) - Apply a client side templete - jquery template.

applyTempServer(value) - Apply a server side templete - mvc partialview.

continueWith(funcontinue) - Tell the javascript funtion to call the first time the plugin receive data.

loadData() - Tell the plugin to start the search.

pageF() - First record.

pageP() - Prev record.

pageN() - Next record.

pageL() - Last record.
